# koai-chat — Design Context

Persistent design context for the `frontend-design` skill and the Impeccable suite (`polish`, `audit`, `critique`, `typeset`, `arrange`, `onboard`, `delight`, `harden`, etc.).

---

## Product

**koai-chat** is the main PWA of KOAI Studios — a multi-agent AI chat interface (Kira + Kronos) with native image generation capabilities, voice input, conversation history, multi-workspace support, and a media gallery. Installable on iOS, Android, and desktop browsers. Production deploy at `chat.koai360.com` (Cloudflare Pages).

It's the daily-driver tool for the KOAI team and eventually for B2B clients who interact with Kira directly.

## Use Cases (jobs to be done)

1. **Chat with Kira** about marketing, strategy, copy, brand guidance, client briefs — text + voice input, streaming responses
2. **Chat with Kronos** about code, architecture, infrastructure — same UX, different model and personality
3. **Generate images** with multiple engines (Gemini for free/fast, Z-Image-Turbo for value, Flux.2 Dev FP8 for premium, Flux Hosted, Studio RAW for advanced) — must show metadata (engine, time, cost)
4. **Voice input** — push to talk, transcribe, edit, send
5. **Manage conversations** — list, search, rename, delete, archive
6. **Browse media gallery** — see all generated images across conversations, filter, view full
7. **System status** — see Kronos brief queue, KOAI mobile briefs, system notifications
8. **Multi-workspace** — switch between client workspaces (each has different context)
9. **Receive push notifications** when long-running tasks finish
10. **Quick switch** between Kira and Kronos agents

## Brand Personality / Tone

KOAI Studios' brand:

- **Confident, premium, creative** — not generic SaaS, not toy
- **Dark-first aesthetic** with neon-accent moments (Kira lime + Kronos cyan)
- **Liquid glass premium** — inspired by Apple Vision OS, Linear, Vercel, motionsites
- **Bilingual mostly Spanish** — UI labels in Spanish, code/tech terms can stay English
- **Confident but not aggressive** — "we know what we're doing" without being loud
- **Functional luxury** — every detail polished, but never gets in the way of work
- **Anti-AI-slop** — no purple-to-blue gradients, no glassmorphism for decoration only, no generic icon-card grids, no "magic sparkles" everywhere

The chat is **production tool first, demo second**. It must feel fast and reliable to use 100x a day, while also being beautiful enough to show off in client demos.

## Visual Identity (already established, do not change)

### Color tokens (defined in `src/index.css`)

| Token | Value | Use |
|-------|-------|-----|
| `--color-bg` | `#0a0a0c` | Main background (dark) |
| `--color-bg-sidebar` | `#111113` | Sidebar |
| `--color-bg-surface` | `#1a1a1e` | Cards, surfaces |
| `--color-bg-elevated` | `#222226` | Elevated cards |
| `--color-text` | `#ffffff` | Primary text |
| `--color-text-muted` | `rgba(255, 255, 255, 0.6)` | Secondary text |
| `--color-kira` | `#D4E94B` | Kira accent (lime/yellow-green) — KOAI brand |
| `--color-kronos` | `#00E5FF` | Kronos accent (cyan) — engineering vibe |
| `--color-purple` | `#7B2D8E` | KOAI brand purple (rare, premium moments) |
| `--color-danger` | `oklch(0.65 0.22 25)` | Errors |

Light mode also exists (`.light` class) — must work in both.

### Typography

- **Display / Headings**: `Sora` (geometric sans, distinctive)
- **Body / UI**: `DM Sans` (humanist sans, very readable)
- **Code / Mono**: `JetBrains Mono`
- **Plus Jakarta Sans, Inter** also loaded as fallback fonts (avoid using Inter as primary — it's the "default AI font")

### Premium effects (already implemented in CSS)

- `.liquid-glass` and `.liquid-glass-strong` — apple-style glass with gradient borders
- `.gradient-text-kira` and `.gradient-text-kronos` — animated shimmer gradients
- `animate-fadeUpBlur` with `stagger-1..6` — staggered entrance animations
- `.grain` — film grain overlay
- Custom cursor with mix-blend-mode on desktop hover devices

### Component library

shadcn/ui pattern with 16 base components (button, dialog, popover, drawer, sheet, command, dropdown-menu, tabs, etc.). All built on Radix UI primitives.

## Stack Constraints

- **React 19** + **Vite 7** + **TypeScript** + **Tailwind 4** (CSS-first via `@tailwindcss/vite`)
- **Framer Motion 12** for animations
- **PWA** real (manifest + service worker custom) — installable
- **Mobile-first responsive** — designed for iPhone/Android first, scales up to desktop
- Must NOT increase bundle size dramatically (current build: 880KB JS, 271KB gzip)
- Must NOT introduce Next.js, Vue, Svelte, or other frameworks
- Must NOT use Tailwind CDN, must NOT use generic shadcn defaults blindly

## Differentiation

What makes `koai-chat` unforgettable vs ChatGPT, Claude.ai, Grok, etc:

1. **Dual-agent interface** — Kira (creative/marketing) and Kronos (technical/code) with **distinct visual identities** (lime vs cyan), not just a model picker
2. **Native multi-engine image generation** — not just one model, you choose the right tool for the job, see the cost, see the time
3. **Liquid glass premium aesthetic** — feels like Apple Vision OS more than SaaS app
4. **Voice + text + image** — true multimodal, not bolted-on
5. **Spanish-first** — not just translated, designed for Spanish-speaking creators
6. **Brand color identity** — Kira lime is unmistakable. Most AI chats are gray-blue.

## Anti-patterns (NEVER do)

Per KOAI Web System v2.0 alertas rojas:

- ❌ Inter as primary font (use Sora + DM Sans)
- ❌ Generic purple-to-blue gradients (instant AI slop signal)
- ❌ Glassmorphism for decoration without purpose
- ❌ Identical icon-card grids (icon + heading + description, repeat 3-4x)
- ❌ "Hero metric" templates (big number + small label + supporting stats)
- ❌ Centering everything — left-aligned with asymmetric layouts feels designed
- ❌ Pure black `#000` or pure white `#fff` (always tinted)
- ❌ Sparkles emojis or "✨ AI ✨" anywhere
- ❌ Bouncing/elastic animations (use ease-out-quart/quint/expo)
- ❌ Cyan-on-dark "developer aesthetic" without purpose
- ❌ Modal dialogs for non-critical interactions (use drawers, popovers, inline)

## When in doubt

When designing a new component for `koai-chat`:

1. First read existing components in `src/components/` to match the established voice
2. Reuse existing CSS utilities (`.liquid-glass-strong`, `.gradient-text-kira`, `animate-fadeUpBlur`, etc.) before inventing new ones
3. Match Sora for headlines, DM Sans for body, JetBrains Mono for code/values
4. Lime-yellow `#D4E94B` for Kira-related, cyan `#00E5FF` for Kronos-related, white/text-muted for neutral
5. Test in both `.dark` and `.light` modes
6. Test on mobile (375px) before desktop
7. Apply `audit` skill after, target P0=0
